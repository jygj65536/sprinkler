## 1. 화면 구성 (라우팅)

`src/App.tsx`의 `screen` 상태로 SPA 전환. 파일 기반 라우팅 아님.

```
screen === 'home'     → Home.tsx     (메인)
screen === 'calendar' → Calendar.tsx  (메인)
screen === 'detail'   → Detail.tsx    (서브)
screen === 'add'      → Add.tsx       (서브)
```

**하단 네비게이션 바**: 홈 / ＋(식물 추가 FAB) / 달력  
- `home`, `calendar` 화면에서만 노출
- FAB(＋) 버튼은 `add` 화면으로 이동

## 2. 건강 상태 판단 로직

```
ratio = daysSince / intervalDays

ratio < 0.7  → healthy  "촉촉해요"    (#5E8C57, 초록)
ratio < 1.0  → thirsty  "곧 목말라요" (#C99A3C, 노랑)
ratio ≥ 1.0  → urgent   "물 주세요!"  (#CC6B52, 빨강)
```

**물주기 타이밍 평가** (Detail 목록·달력에서 사용):
```
gap = 이번 물주기 - 직전 물주기 (일 수)

gap ≤ intervalDays × 1.15 → "제때"      (#5E8C57)
gap ≤ intervalDays × 1.5  → "조금 늦음" (#C99A3C)
gap > intervalDays × 1.5  → "많이 늦음" (#CC6B52)
첫 번째 물주기             → "첫 물주기" (레이블 없음)
```

**다음 물주기 예정일**:
```
dueDate = 마지막 물주기 날 + intervalDays
daysUntil = dueDate - today

daysUntil < 0  → "{N}일 지났어요" (빨강)
daysUntil = 0  → "오늘이 그날!"   (빨강)
daysUntil ≤ 2  → "{N}일 뒤"       (노랑)
daysUntil > 2  → "{N}일 뒤"       (회색)
```

## 3. 홈 화면 (`Home`)

### 표시 정보
- 오늘 날짜 (YYYY년 M월 D일 요일)
- **요약 배너**: 물이 필요한 식물 수 표시
  - 1개 이상: "{N}개의 식물이 물을 기다려요"
  - 0개: "모두 촉촉해요"
- **선반 뷰**: 식물 3개를 한 선반에 배치. 식물 클릭 시 Detail 이동
  - 각 식물 아래 이름 + 건강 상태 뱃지 (색 점 + 레이블)
- **빈 상태**: 식물 0개일 때 안내 메시지

### 액션
| 액션 | 조건 | 결과 |
|------|------|------|
| 식물 카드 탭 | — | Detail 화면으로 이동 |
| "새로운 식물 들이기" 버튼 | — | Add 화면으로 이동 |

## 4. 달력 화면 (`Calendar`)

### 표시 정보
- 월 이동 (‹ / ›)
- **식물 필터 칩**: 내 식물 목록에서 토글. 체크된 식물만 달력에 반영
- **달력 그리드** (일~토, 7열)
  - **과거 점 (채워진 원)**: 물 준 날에 식물 고유 색 점 (최대 4개)
  - **미래 점 (빈 원)**: 식물이 2개 이상 선택 시, 각 식물의 예정일에 색 테두리 원
  - **구간 밴드**: 식물이 1개만 선택 시, 예정일 ±2일 범위를 반투명 배경으로 표시
  - **중심 점**: 식물이 1개만 선택 시, 정확한 예정일에 테두리 원 강조
  - 오늘: 원형 강조 표시
  - 일요일: 빨강 날짜 숫자
- **범례**: 물 준 날 / 물 줄 날 / 물주기 구간
- **다가오는 물주기 리스트**: 선택된 식물의 예정일을 오름차순 정렬
  - 식물명 / 추천 구간 (예정일 ±2일) / 예정일까지 D-day 텍스트
  - 탭 시 Detail 이동

### 액션
| 액션 | 결과 |
|------|------|
| 식물 칩 토글 | 달력·리스트 필터링 |
| ‹ / › | 월 변경 |
| 다가오는 물주기 행 탭 | Detail 이동 |

## 5. 내 식물 상세 화면 (`Detail`)

### 표시 정보
- 식물 일러스트 (SVG 두들)
- 이름 (커스텀명) + 학명
- 건강 상태 뱃지
- **유대 카드** (2칸):
  - 함께한 지 N일 (등록일 기준)
  - 마지막 물주기 N일 전 + 다음 물주기 D-day
- **이 아이가 좋아하는 것**: 빛 / 물 줄 때 / 온도 / 물주기 / 물 양
- **물주기 일기** (목록 / 달력 토글):
  - **목록 뷰**: 역순 (최근 순), 날짜 + 요일 + 간격 태그 + "{N}일 만에 줬어요"
    - 오늘 기록이 있으면 최상단 항목에 "취소" 버튼 노출
  - **달력 뷰**: 현재 선택된 월 기준 미니 캘린더. 물 준 날에 타이밍 색 원 표시

### 액션
| 액션 | 조건 | 결과 |
|------|------|------|
| "물 줬어요" 버튼 | 오늘 미기록 | wateringLogs에 today 추가, 토스트 표시 |
| "물 줬어요" 버튼 | 오늘 이미 기록 | 토스트 "오늘 이미 기록됐어요" |
| "물 줬어요" (과습 감지) | ratio < 0.4 | 토스트 "과습 주의 💧" |
| "취소" 버튼 | wateringLogs[-1] === today | 오늘 기록 제거, 토스트 |
| "내 식물에서 보내주기" | — | 식물 삭제 후 Home 이동, 토스트 |
| ‹ 뒤로 | — | 이전 메인 화면(home/calendar)으로 복귀 |

## 6. 식물 추가 화면 (`Add`)

### 플로우
1. **검색**: 이름(한글) 또는 학명(영문 대소문자 무관)으로 SPECIES_DB 필터링
2. **결과 리스트**: 식물명 / 학명 / 물주기 간격 표시. 탭 시 바텀 시트
3. **바텀 시트** (종 상세 미리보기):
   - 일러스트 + 이름 + 학명
   - 식물 설명 텍스트
   - 케어 정보: 빛 / 물 줄 때 / 온도 / 물주기
   - **이름 지어주기** 입력 필드 (기본값: 종 이름)
   - "내 식물로 들이기" 버튼
4. **등록 완료**: wateringLogs = [today]로 초기화, Home으로 이동, 토스트

### 엣지 케이스
- 검색어 있고 결과 0건: "아직 우리 정원에 없는 친구예요 🌱"
- 검색어 없을 때: 예시 힌트 텍스트 표시

## 7. 공통 UX

### 토스트
- 하단 네비게이션 위 fixed 위치
- 1.9초 후 자동 사라짐
- 동일 타이머 재사용 (연속 액션 시 갱신)

- 상단: `max(62px, calc(44px + var(--safe-top)))` (메인), `max(54px, calc(38px + var(--safe-top)))` (서브)
- 하단 네비: `calc(84px + var(--safe-bottom))`

### 디자인
- TDS 미사용, 자체 SVG 두들 기반 일러스트 UI
- 라이트 모드 전용
- 폰트: Shantell Sans (본문), Caveat (제목·숫자 강조)
