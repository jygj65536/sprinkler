# 물주기 알림 정책

## 1. 개요

식물의 마지막 물주기 날짜 + 권장 주기를 기반으로, 물주기 당일 오전 9시에 토스 푸시 알림을 발송한다.

---

## 2. 핵심 제약

- **AIT Smart Message API는 서버 간 통신 전용** (mTLS 필수)
- 클라이언트 앱은 알림 동의 UI 수집과 스케줄 데이터 전송만 담당
- 실제 푸시 발송은 백엔드 서버(Supabase Edge Function)에서 처리

---

## 3. 전체 아키텍처

```
앱 (클라이언트)                    Supabase Edge Function         AIT 서버
  │                                       │                          │
  ├─ 앱 초기화                             │                          │
  │   └─ getAnonymousKey() → userKey 저장 │                          │
  │                                       │                          │
  ├─ 첫 식물 추가 완료 후                  │                          │
  │   └─ requestNotificationAgreement()  │                          │
  │       └─ 동의 시 → upsert-schedule ─→ │ (스케줄 저장)             │
  │                                       │                          │
  ├─ 물주기 기록 시                        │                          │
  │   └─ upsert-schedule ───────────────→ │ (다음 물주기 날짜 갱신)   │
  │                                       │                          │
  ├─ 식물 삭제/보관 시                     │                          │
  │   └─ delete-schedule ───────────────→ │ (스케줄 삭제)             │
  │                                       │                          │
  │                               매일 KST 09:00 (cron)              │
  │                                       ├─ next_watering_date = 오늘인 스케줄 조회
  │  ←── 토스앱 푸시 알림 ────────────────┤── POST /send-message ───→ │
```

## 4. 알림 발송 정책

| 항목 | 내용 |
|------|------|
| 발송 시점 | 물주기 당일 오전 9시 (KST) |
| 대상 | 알림 동의한 유저 중 당일 물주기 예정인 식물 보유자 |
| 메시지 형식 | `[식물이름]에 물 줄 시간이에요 🌿` |
| 복수 식물 | 식물마다 개별 발송 (1식물 = 1메시지) |
| 물주기 날짜 계산 | `마지막 물주기 날짜 + 현재 계절의 waterIntervalDays` |

### 계절별 물주기 주기 적용

서버에 스케줄 저장 시 **next_watering_date를 클라이언트가 계산해서 전달**한다. 계절 판단은 클라이언트(앱)에서 수행하며, 물주기를 기록할 때마다 재계산해서 서버에 갱신한다.

```
next_watering_date = 마지막 물주기 날짜 + waterIntervalDays[현재 계절]
```

## 5. 알림 동의 수집

### 동의 요청 시점

- **첫 식물 추가 완료 직후** (홈 화면으로 돌아온 후 바텀시트 노출)
- 강제 팝업 금지 (AIT 검수 탈락 사유 — 자연스러운 맥락에서만 요청)

### SDK 함수

```ts
import { requestNotificationAgreement } from '@apps-in-toss/web-framework';

const cleanup = requestNotificationAgreement({
  options: { templateCode: TEMPLATE_CODE }, // AIT 콘솔에서 발급
  onEvent: async ({ type }) => {
    cleanup();
    if (type === 'newAgreement' || type === 'alreadyAgreed') {
      await syncAllSchedules(userKey, plants);
    }
  },
  onError: (e) => { cleanup(); },
});
```

### 동의 결과 처리

| 결과 | 처리 |
|------|------|
| `newAgreement` | 전체 식물 스케줄 서버 일괄 등록 |
| `alreadyAgreed` | 전체 식물 스케줄 서버 일괄 등록 (최신화) |
| `agreementRejected` | 아무것도 하지 않음 (재요청 금지) |

## 7. 서버 데이터 모델 (Supabase)

### `notification_schedules` 테이블

```sql
CREATE TABLE notification_schedules (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_key            text NOT NULL,
  plant_id            text NOT NULL,
  plant_name          text NOT NULL,
  next_watering_date  date NOT NULL,
  created_at          timestamptz DEFAULT now(),
  updated_at          timestamptz DEFAULT now(),
  UNIQUE(user_key, plant_id)
);

CREATE INDEX idx_next_watering_date ON notification_schedules(next_watering_date);
```

**저장 범위**: 알림 스케줄만 저장. 식물 전체 데이터는 기기 로컬(NativeStorage)에만 있음.

### 스케줄 업데이트 시점

| 이벤트 | 동작 |
|--------|------|
| 알림 동의 완료 | 모든 활성 식물 일괄 upsert |
| 물주기 기록 | 해당 식물 upsert (next_watering_date 갱신) |
| 물주기 취소 | 해당 식물 upsert (next_watering_date 롤백) |
| 식물 삭제 | 해당 식물 스케줄 delete |
| 식물 보관 | 해당 식물 스케줄 delete |
| 식물 보관 해제 | 해당 식물 upsert (알림 동의 상태일 때) |

## 8. Edge Functions

### `upsert-schedule` — 클라이언트 호출용

- **Method**: POST
- **Auth**: Supabase Anon Key (클라이언트에서 Authorization 헤더)
- **Body**: `{ userKey, plantId, plantName, nextWateringDate }`

### `send-daily-notifications` — cron 실행용

- **Schedule**: `0 0 * * *` (UTC 00:00 = KST 09:00)
- **동작**:
  1. `next_watering_date = 오늘`인 레코드 전체 조회
  2. 각 레코드마다 AIT `/send-message` API 호출 (mTLS 인증)
  3. 성공/실패 로깅

## 9. AIT 콘솔 선행 작업 (코드 작업 전)

| 작업 | 경로 | 결과물 |
|------|------|--------|
| 알림 동의문 등록 | 콘솔 → 스마트 발송 → 알림 동의문 탭 | `templateCode` |

| mTLS 인증서 발급 | 개발자센터 → API 사용하기 | 인증서 파일 |
