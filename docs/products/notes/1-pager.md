# 0. background

- 토스 앱인토스에 입점할 미니앱을 만든다
- 미니앱의 조건은
    - 너무 짜치지 않고 (광고 덕지덕지 only는 X)
    - 내가 실제로 일상에서 쓸 수 있을 퀄리티로 만들되
    - 리소스를 지나치게 많이 쓰지 않도록 한다

# 1. overview

## Objective

식집사들이 식물을 키울 때 물주기 타이밍을 잊지 않게 해, 꼭 참고할 수 있는 앱이 된다

### 1) Problem

- 식물을 키울 때 언제 물을 줘야하는지 판단하기 어렵다
- 식물을 키울 때 내가 언제 과거에 물을 줬는지 기억하기 어렵다

### 2) Opportunity

- 식물을 키울 때 언제 물을 줬는지 꾸준히 기록하게 한다
- 과거 물주기 타이밍을 기반으로, 물을 줘야하는 타이밍에 알림을 보내 잊지 않게 한다
- → 유저의 식물을 살리면, 유저는 앱을 쓸모있다고 여길 것이고, 이탈하지 않고 앱을 계속 사용하게 될 것이다

### 3) Needs

- 내가 가지고 있는 식물들에 대한 정보를 알려주고, 새로운 식물이 생겼을 때 편하게 추가할 수 있게 한다
    - 식물들의 이름을 검색하면 자동으로 그 식물의 기본적인 정보와 사진, 이상적인 물주기 기간등을 데이터베이스에서 불러오고 조회할 수 있게 한다
    - 식물들의 이름을 검색한 뒤, “내 식물” 로 추가하게 되면 “내 식물 목록” 에서 해당 식물을 조회하고 관리할 수 있게 한다
- 내가 식물들을 과거에 관리한 내역들을 꾸준히 기록하고 조회할 수 있게 한다
    - 물 준 날짜를 캘린더 & 기간 기반으로 저장하게 한다
    - “내 식물” 하나 당 언제 물을 줘왔는지 과거 내역을 조회할 수 있게 한다
        - 해당 과거 내역을 노출하는 방식은 list & calendar 크게 두 가지의 뷰가 가능할 것으로 보임
- 내가 과거에 식물을 관리한 내역을 기반으로, 언제 물을 줘야하는지 알람을 보내주게 한다
    - 그 식물 데이터베이스에서 불러온 이상적인 물주기 기간을 바탕으로, 마지막으로 물 준 날짜에서 해당 기간이 지났다면 유저에게 알림을 보내도록 한다
        - 알림의 방식은 크게 아래와 같다
            1. 앱 접속 시 메인화면
            2. 푸시알람
            3. 위젯에서의 화면 변경

# 2. 기능 개요

크게 세 가지의 화면으로 구성된다

## `(main) Home`

- 내 식물 목록 확인
    - 물을 많이 줬는지, 덜 줬는지에 따라서 식물의 건강상태가 직관적으로 보이게 한다
- 새로운 식물 추가

## `(main) 달력`

- 달력에 노출하는 식물의 수는 유저가 선택해서 조절할 수 있게 한다
- 과거 - 식물당 물주기 내역
    - 건강하게 물주기 했는지 여부를 색깔로 바로 직관적으로 파악할 수 있게 알려준다
- 미래 - 식물당 물 줘야하는 날짜 알림
    - exact할 필요 없고, 생리주기처럼 물주기 가능한 날짜 range로 표기하기 → 시각적 복잡성 고려필요

## `(sub) 내 식물 상세`

- 내 식물에 대한 메타정보
    - 식물이 잘 자라는 환경
    - 이상적인 물 주기
- 나와의 유대 정보
    - 언제 최초로 등록했는지
    - 등록한지 얼마나 되었는지 (함께한지 얼마나 되었는지)
- 물주기 데이터
    - 마지막으로 물 준지 얼마나 경과했는지
    - 과거에 물 줬던 이력 조회
    - 필요한 물주기 양

## `(sub) 식물 추가하기`

- 식물명 검색
- → `식물 상세정보 조회`
- → 내 식물로 추가하기
